# 回归验收清单（可复跑）

> 用途：把《申论练习器开发计划》里所有「尚未完成 / 待验收」流程收拢成一份**可重复执行**的验收单。每次改动 `规范表达练习.html` 后按本单从头跑一遍。
>
> 权威计划见 `申论练习器开发计划.md`；本单只负责“怎么验、判据是什么、谁来验”。

## 由谁验、用哪条入口

| 角色 | 负责范围 | 入口 |
| --- | --- | --- |
| **阿楠（真机/真实浏览器）** | 触摸选择、输入法、真实手机视口、AI 真配置批改 | 桌面双击 `启动规范练习.bat` → 浏览器打开 `http://localhost:8080/规范表达练习.html`；手机浏览器打开同一文件 |
| **开发者（静态/无头）** | 语法、静态状态截图、向导各步入口、离线降级、暗色/窄屏 | `node --check`（见下）+ 打开 `_dev/_shot.html?...` 各场景 |

**为什么用 `http://` 不用 `file://`**：FSA / IndexedDB 在 `file://` 下行为不同且历史上会卡启动。无头验证一律 `python -m http.server`（`.bat` 已封装）。

### 三条固定入口

1. **静态自检**（开发者，项目根目录运行）：
   - `node _dev/check.js` — 抽取内联 `<script>` 跑 `node --check`（语法）+ 校验每个 `$('#id')`/`$$('#id')` 都能解析到文档里的 id。退出码 0 = 全过。
   - `node _dev/_validate.js` — 校验 `SUMMARY_BANK` 切片拼接、`point` 索引、必填字段（ERROR 令退出码 1）。
2. **人工全流程**：`启动规范练习.bat` → `http://localhost:8080/规范表达练习.html`。
3. **场景截图台**：`_dev/_shot.html?act=<场景>`（参数见文件头注释）。产物截图落 `_dev/shots/`。

> 状态口径（阿楠敏感项）：报告时必须区分 **已写入代码 / 已过语法检查 / 已浏览器实测 / 尚未实测**。本单每项的“实测”指在真实浏览器里点过一遍。
